2. **The oldest host is not the origin.** A personal knowledge-assistant project (repo init 2025-01-09) is the family's oldest repo, but its framework material *arrived* by migration: a memento migration plan and framework-named tooling appear from 2025-07-05 (`c5b0459` "implement unified Memento Framework search command") — received, not born there.
3. **The client-programme assistant was born WITH the framework** (init commit 2025-07-19, message verbatim save the redacted name: "initialize … with Memento framework" — US spelling is the source's) — a recipient three weeks after the origin repo's memento structure, which then became the most mature deployment.
4. **The extraction drew on all three.** The User's own extraction-era working notes (`~/MEMENTO_EXTRACTION_FOCUS.md`, dated 2025-08-07, the eve of the canon's first commit) state the canon was created "from three production implementations" — the client-programme assistant (primary), the roadmap tool v2, and the knowledge assistant — staged through a working directory and sanitised for release.
5. **Consequence for the canon's story:** the strongest continuity credential holds in its strongest form and is receipted — *the repo where the framework was born is still running the framework thirteen months later*, and this audit read it live. The first cross-pollination events date to July 2025 (origin → knowledge assistant migration; origin-pattern → client deployment), a year before the current wave.
